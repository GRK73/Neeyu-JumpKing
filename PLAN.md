## 두 루트 구조

```
Route A (왼쪽, x=20~420)   Route B (오른쪽, x=540~940)
    ↓ 각 스테이지 내부 독립 진행
    ↓ 스테이지 상단 Connector (x=380~580) 에서 합류
    ↓ 다음 스테이지에서 다시 분기
```
- 두 루트 모두 동일한 스테이지를 거침
- Connector 플랫폼 폭 200px: 양쪽 루트가 도달 가능

## 세이브포인트 (스테이지 5)
- `SAVE_POINT_Y = 3200` (스테이지 5 하단)
- 플레이어가 y < 3200–CHAR_H 이상 올라가면 `savepointUnlocked = true`
- 활성화 후 낙하 시 `py + CHAR_H ≥ SAVE_POINT_Y` → 스테이지 5 바닥에 착지
- 구현: y=3200에 invisible savepoint 플랫폼 (savepointUnlocked=true일 때만 충돌)

## 기믹 구현 스펙

### 부서지는 바닥 (스테이지 3)
- 플랫폼 속성 `gimmick:'breakable'`
- `standTimer` 누적 → 5000ms 초과 시 `broken=true`
- broken 플랫폼 충돌 무시, 10초 후 리스폰
- 시각: standTimer 진행에 따라 회색→노랑→주황→빨강 색상 변화

### 선풍기 (스테이지 4)
- 별도 `FANS` 배열: `{ x, y, w, h, windX, windY, range }`
- state='start'/'end' 일 때 range 내에 있으면 vx/vy에 힘 가산
- 힘의 세기: `strength × (1 – dist/range)` (거리 기반 감쇠)
- 시각: 팬 아이콘 + 바람 방향 화살표

### 얼음 바닥 (스테이지 7)
- 플랫폼 속성 `gimmick:'ice'`
- 착지 시 vx 유지 (0으로 만들지 않음), vy=0만
- 서있는 동안 vx *= 0.97 (마찰 감소)
- 이동 입력: 가속도 방식 (즉각 이동 대신 0.4씩 가속)
- 시각: 밝은 파란색 + 눈결정 아이콘

### 튕기는 함정 (스테이지 6)
- 별도 `TRAPS` 배열: `{ x, y, w, h, bounceVX, bounceVY }`
- 충돌 시 vx/vy를 bounce 값으로 교체 → 'start' 상태 진입
- 시각: 빨간 마름모 / 스프링 모양

### 이동 플랫폼 (스테이지 8)
- 플랫폼 속성 `gimmick:'moving'`
- 추가 속성: `moveAxis, moveMin, moveMax, moveSpeed, moveDir`
- 매 프레임 위치 업데이트 + 위에 서있는 플레이어도 함께 이동
- 시각: 플랫폼 위에 이동 방향 화살표

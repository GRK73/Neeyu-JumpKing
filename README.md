# Neeyu Jump King

## 프로젝트 개요
Neeyu Jump King은 순수 JavaScript와 HTML5 Canvas API를 사용하여 개발된 웹 기반 플랫포머 게임입니다. 이 게임은 여러 테마의 스테이지로 나뉜 연속적인 수직 맵을 특징으로 하며, 자체적인 물리 엔진, 충돌 감지 및 다양한 상호작용 가능한 스테이지 기믹을 구현하고 있습니다.

## 기술 사양
- **해상도:** 기본 960x540 해상도(16:9 비율)를 기반으로 창 크기에 맞춘 동적 스케일링 지원
- **기술 스택:** HTML5, CSS3, 순수 JavaScript (ES6 모듈)
- **렌더링:** HTML5 `<canvas>` API 및 성능 최적화를 위한 오프스크린 캔버스 활용
- **환경:** 최신 웹 브라우저 환경에서 동작하며, GitHub Pages 배포에 적합한 구조

## 프로젝트 구조
```text
neeyu_jumpking/
├── index.html          # 메인 게임 진입점
├── map_editor.html     # 전용 맵 에디터 도구
├── package.json        # 프로젝트 메타데이터
├── PLAN.md             # 초기 게임 기획 및 맵 구조 계획
├── willupdate.md       # 렌더링 및 텍스처 구현 계획
├── style.css           # 전역 스타일
├── assets/
│   └── sprites/        # 캐릭터 애니메이션 프레임 (stand, run, ready, start, end)
└── src/
    ├── constants.js    # 전역 상수 (예: 셀 크기)
    ├── engine.js       # 핵심 게임 루프, 물리 엔진, 충돌 논리
    ├── main.js         # 엔진, 렌더러, 입력을 연결하는 진입점
    ├── map_data.js     # 맵 데이터, 플랫폼 정의 및 스케일링 함수
    └── renderer.js     # 캔버스 렌더링 로직 및 스프라이트 관리
```

## 핵심 기능

### 1. 물리 및 게임 엔진
- 중력, 점프 역학 및 속도를 처리하는 자체 제작 물리 엔진
- 정밀한 AABB(Axis-Aligned Bounding Box) 충돌 감지
- 창 크기에 맞춰 동적으로 조정되는 스케일러블 맵 좌표계
- 차지 점프 및 이동 제어를 지원하는 키보드 입력 처리

### 2. 맵 시스템 및 스테이지
맵은 총 7000 단위의 수직 높이로 구성되며 여러 스테이지로 구분됩니다.
- **Floor (6440-7000):** 시작 지점
- **Stage 1 (5600-6440):** 돌 바닥 / 무대 단상
- **Stage 2 (4800-5600):** 빨간 천 바닥 / 빨간 커튼
- **Stage 3 (4000-4800):** 철 바닥 / 직원 사무실
- **Stage 4 (3200-4000):** 빨강·금 빔 / 서커스 천막 꼭대기
- **Stage 5 (2400-3200):** 땅 바닥 / 숲 (세이브포인트 존재, Y=3200)
- **Stage 6 (1600-2400):** 나뭇가지 / 숲 상층 및 하늘
- **Stage 7 (800-1600):** 구름 / 하늘
- **Stage 8 (0-800):** 운석 / 밤하늘
- **Goal (~40):** 최종 목표 지점

맵은 양쪽으로 진행할 수 있는 두 개의 루트(왼쪽 Route A, 오른쪽 Route B) 구조를 가지며 특정 플랫폼(Connector)에서 합류합니다.

### 3. 상호작용 기믹
- **부서지는 바닥 (스테이지 3):** 5초 동안 밟으면 부서지고 10초 후 재생성되는 플랫폼
- **선풍기 (스테이지 4):** 접근 거리에 비례하여 지정된 방향으로 플레이어를 밀어내는 기믹
- **얼음 바닥 (스테이지 7):** 마찰력이 감소하여 착지 및 이동 시 미끄러지는 플랫폼
- **튕기는 함정 (스테이지 6):** 플레이어의 속도를 변경하여 지정된 방향으로 튕겨내는 기믹
- **이동 플랫폼 (스테이지 8):** 정해진 축을 따라 이동하며 탑승한 플레이어를 함께 이동시키는 플랫폼

### 4. 렌더링 및 최적화
- **오프스크린 캔버스 텍스처:** 플랫폼 텍스처와 배경은 오프스크린 캔버스에 미리 렌더링된 후 `drawImage`를 통해 재사용되어 안정적인 60 FPS를 유지합니다.
- **시차 스크롤 배경:** 카메라 이동 속도의 일부만 적용되어 스크롤되는 배경으로 원근감을 제공합니다.
- **스테이지 전환 효과:** 스테이지 환경 전환 시 알파 트윈을 통한 부드러운 페이드 효과를 적용합니다.

## 설치 및 실행 방법

1. 저장소를 클론하거나 다운로드합니다.
2. 로컬 웹 서버를 사용하여 디렉토리를 서빙합니다. (예: `npx serve`, Live Server 확장 프로그램, 또는 Python의 `http.server`)
   ```bash
   # Python 3을 사용하는 예시
   python -m http.server 8000
   ```
3. 최신 웹 브라우저를 열고 `http://localhost:8000/index.html`에 접속합니다.
4. 맵 에디터는 `http://localhost:8000/map_editor.html`을 통해 접근할 수 있습니다.
